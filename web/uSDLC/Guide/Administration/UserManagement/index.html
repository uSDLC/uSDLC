User Management

For all except singular use, uSDLC provides user management to control page access and provide page history.

Authentication

Once an account has been created, a user can log in from the main menu item User/Log in.

Logging out will reset the system to the default user for the current installation. In most cases, this is Guest.

Because uSDLC is designed to be used on individual machines for development and test groups, it is possible to set the default use at launch time. Modify uSDLC.sh or uSDLC.bat and change the launch line to include

... 'userId=My Name' ...

where the quotes ensure that spaces in the user name do not effect the launch. For unix based systems, a \ in front of the space will work.

Administrator

On a default installation, neither Guest nor Administrator have a password. Log in as Administrator to create your own user. You can change the Administrator password at the same time. An administrator has full access to all pages regardless of group membership.

Guest

The default user, or logging out, make the current user Guest. A Guest can only see the public pages - and can not change anything.

Creating a new user

Go to the primary project for this user and create a new page under Environment / Users with the same name as the user. In the new page, create at least two sections referencing the instrumentation files Login.groovy and Password.txt.

Leave Password.txt empty. When you first log in you do not need to provide a password. If this is a shared copy, change the password. As you will see, the contents saved are encrypted.

Login.groovy

This is a simple DSL providing basic user information (name and email), what groups the user has access to and at what level.

name 'Fred Flinstone'
email 'fred@boulder.com'

read access
groups 'public'

write access
groups 'projectA-design projectB-dev'

execute access
groups 'projectC-dev'

Groups

Any page can have a reference to a Groups.csv file - usually in the footer. Access restriction for that page and all children (until the next Groups.csv) is based on the groups and access type for the currently logged in user.

Look at the main uSDLC page in the list at the left. Groups are defined at the bottom of the page - public and usdlc-developers. As Guest belongs to the public group with read access, even a logged out user can see this page. A user with write access to usdlc-developers can make change as well as gain access to other pages of that group that do not have public.
