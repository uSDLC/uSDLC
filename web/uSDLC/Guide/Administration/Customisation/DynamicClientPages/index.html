Dynamic Client Pages

Whenever a page is loaded into the uSDLC framework, elements are scanned for attributes. This can be used to load html content, run server-side scripts and/or run pre-loaded script methods.

Dynamic HTML Loading

If a <div> element has a href attribute and no action attribute, or one called setInnerHtml, the the page requestes is loaded from the server to within the <div>.

Use an extension of htm to avoid the uSDLC parsing of html.

e.g.

<div href='DynamicHTMLLoading.htm'></div>

and again as HTML code...

Dynamic Script Loading

uSDLC supports client side scripts in Javascript or Coffeescript. Use a <div> element with a href attribute pointing to the script and an action attribute of globalEval.

e.g.

<div href='DynamicScriptLoading.coffeescript' action='globalEval'></div>

and again as HTML code...

Dynamic Content Loading

The action attribute is a reference to a function in the usdlc name-space. When the <div> element is parsed, the method is called passing the element and the data retrieved. The menu is a structured text file. It is loaded with

<div href='/rt/top.menu' action='loadMainMenu'></div>

triggering a call to the code in menu.coffeescript.

Element Activation

Finally any element can have an attribute called activate, the value of which is expected to be a method in the usdlc name-space. It is run when the page is loaded - and passed the enclosing element.

e.g.

<div activate="green">Make this section green</div>

and again as HTML code...
