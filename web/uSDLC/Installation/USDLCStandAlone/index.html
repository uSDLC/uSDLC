uSDLC Stand-Alone

No install - just run

The Single Install Model

If you have a Java jre installed you can download a single executable uSDLC.jar that will unpack and prepare itself for use. Optional components, languages and systems are downloaded as required.

For Windows, If you don't have Java installed then you can download a bootstrap loader. When run it will create the environment needed by uSDLC, download the executable jar and run it. All Unix varients will have a version of java available. "java -jar usdlc" is enough to unpack and run uSDLC.

Local configurations and optional components need only be set up on one machine. Additional machines once added to the uSDLC web will update themselves as needed.

Running the Server

Once the system has been unpacked you have access to the start script in the uSDLC root directory. Just type uSDLC on the command line and the server will start using default values.

If the uSDLC server is already running on the same port, the running instance is stopped before the new one starts. This is useful if you change libraries or other cached code or data.

Running the Client

uSDLC is browser based. It should work correctly from any modern offering that supports JQuery. When the server starts it gives the host address to use. Using this address the server can be accessed from any machine on the same network.

The browser can be pointed to any page in the document by giving it's full address. This address can be copied to the clipboard by right-clicking on the page title and saving it from the context menu. By default the page will open in a uSDLC decoration page. If, however, the command line option action=run is added, the page will be run after loading.

Headless Operations

uSDLC can be treated as a service by adding the command line option mode=server. Headless operation means that a browser is not needed to perform the operations. The default response is HTML. Use the format option:

  1. format=html: Output will be sent to the browser as it is created. It will look like plain human-readable text with some decorations.
  2. format=xml: Output is in XML for consumption by another program - such as a contiuous build system.
  3. format=text: Output is plain text. Useful if running with curl or save to a file.
