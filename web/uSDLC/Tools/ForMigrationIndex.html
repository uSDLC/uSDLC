Tools for Migration

Migration is not just data

In the industry the term used is "data migration" with the implied expectation that the new software version will be deployed at the same time. In practice it makes sense to have all migration activities in one place.

Miration includes:

  1. Partial or full application deployment
  2. Data table creation, inserts, deletes, updates and format transfers.
  3. Configuration changes similar to those needed for the database

Migration Protection

  1. Confirm pre-conditions
  2. Confirm the absense of exclusions (as in already installed)
  3. Confirm correct results after process

Requirements

  1. Table diff tool to extract migration specific updates
  2. File/director diff tool to extract migration specific changes
  3. Edit and Run pure SQL
  4. Accept CSV, spreadsheet native and XML data for table creation or update
  5. Upload to uSDLC and allow direct edit from a link
  6. Easy to use DSL for where more logic is required in the migration
  7. DSL includes features for text file updates for properties
  8. Run any of the above on a remote machine
