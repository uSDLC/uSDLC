uSDLC Tools for Developers

Making my life easier

Developers live by their tools. This section lists tools that help them use their other tools.

Curl

This is not really a tool but a convenience function. Developers are encouraged to write the tests in the uSDLC system prior to writing the production code. These tests are embedded in the documentation. For convenience it is better to be able to run the tests from the IDE or command line. Fortunately this is easy with groovy.

Select copy url from the context menu when hovering over an actor, section or page heading. Edit Curl.groovy in the uSDLC web root and paste in the URL. Now you can run Curl from your IDE or command line and see the results without having to switch to a uSDLC window.

Logging

Your application may have a complete logging system. Your actors only need a more basic tool. Anything written to the standard error stream will display on the uSDLC server console. For convenience, Groovy and Groovy based DSLs can call log.

Timings

Use the Timer class if you want to measure how long a task takes. The time is always measured from instantiation. The toString() method returns the elapsed time into human readable form - hours, minutes and seconds or milliseconds. Timings can also be recorded to a long for later review.
