Page Updating

Creating and editing uSDLC pages on-the-fly

uSDLC pages are created, updated and deleted from the user interface. There are times when it is useful to do the same operations from server code. Screencast creates a new page to build a new screencast on. Tasks and server-side runs update instrumentation links to record changes.

Opening a page gives you access to the title, subtitle, synopsis and sections.

Creating a new page happens by asking for it by name. The name is mashed up to create the title. The subtitle and synopsis are left empty, but you can set them as you please. Note that the page reference uses a Store object rather than a string. Either constructor can be used for creation and opening of pages.
