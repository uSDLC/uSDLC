Authorisation

Who can do what when

Authentication and authorisation are DSL based - using a page per user and an authorisation script per page.

Authentication

Users log in and out from any page they have access to by selecting from the menu.

Every user has a home page under 'Environment / Users'. A user can view but not update this page. Any scripts on the page are run on log-in.

Every page can have a groups section. Only admin and users who are members of the group may access the page. Group memberships can be given read, write or instrument status.
