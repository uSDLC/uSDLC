Actors

Scripts that know how to give uSDLC what it needs

Actors are scripts or static files that send data back to the uSDLC client that control and inform. Any text is displayed in a box connected to any running links. 

Transport

Actors receive all information from standard input and can send all results as HTML to standard out.

Input

Actors read from standard input (stdin). They can read until end-of-file which is when standard input is closed by the server. Apart from the command specific input, there is also a dictionary of environment specific header information. How this is passed depends on the actor. For Groovy scripts (and all JSR-223 compliant languages), for example, it is in a global called env. For true CGI scripts it is in the operating system environment variables.

Header Information

Standard input is reserved for script data. Environmental data is also provided in a script specific fashion as a dictionary of name/value pairs.

  • accept: application/xml,application/xhtml+xml,text/html;q=0.9,text/plain;q=0.8,image/png,*/*;q=0.5]
  • acceptencoding: gzip,deflate,sdch
  • acceptcharset: ISO-8859-1,utf-8;q=0.7,*;q=0.3
  • acceptlanguage: en-US,en;q=0.8
  • clientType=html
  • cookie: name=value;second=secondvalue;etc=more
  • fragment: after # on address line
  • host: 127.0.0.1:9000
  • method: GET
  • querystring: name=valye&etc=more
  • script: script-name
  • referrer: uri-of-referring-page
  • uri: /uSDLC/index.html
  • useragent: Mozilla/5.0 (Windows; U; Windows NT 6.1; en-US) AppleWebKit/534.10 (KHTML, like Gecko) Chrome/8.0.552.224 Safari/534.10]

Cookies have some uSDLC-specific parameters set:

  • session: unique id for this browser session
  • user: Id for user logged in or anon

Output

The server and browser both expect HTML output. Script tags are interpreted and the rest recorded and displayed.

Response

The active components are inside script tags (e.g. <script>usdlc.action()</script>). You can run any Javascript or Jquery command in addition to the uSDLC ones supplied. Every actor should include either usdlc.passed() or usdlc.failed().

  • usdlc.passed() - Call to mark actor as working correctly.
  • usdlc.failed() - Call to mark actor as failing to work as expected.
  • usdlc.alert(messagePage) - Display an alert message given in a file. If it ends in .htm then it will reside in /rt/alerts/*.htm, otherwise it can be in a page created within this system - with one or more sections. This link will display a sample alert.
  • usdlc.highlight(colour) - Put a coloured strip on the left of the current section. Colour can be blue, brown, green, pink, red, sky or yellow.
  • usdlc.runPage() - Run the whole page as if the user had clicked the play button on the tool-bar
  • usdlc.runFromSectionInFocus() - Run the highlighted section and all those below
  • usdlc.runSectionInFocus() - Run the highlighed section
  • usdlc.setEditMode(true|false) - Change the page mode so that the default is edit or run on double-clicking a section.
