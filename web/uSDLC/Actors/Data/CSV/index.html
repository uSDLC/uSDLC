CSV Data Files

The lowest common data denominator

CSV stands for Comma Separated Variables. All spreadsheets and most database clients can be asked to create CSV files. There is no standard. Comma is common, but so is tab and column alignment (COBOL). The data may contain the separator, so a mechanism to quote each variable is used. This is most commonly single or double-quotes, but percent is also used. Lastly, some mechanisms allow a quoted variable to include multiple lines.

Uploading a CSV File

Select the upload menu option for a highlighted selection. As with all uploads, a copy will be placed in the current section. If the file is new a link will be added to the end of the section. If the file selected has a csv extension, it's contents will be displayed. The links at the end of this section was created with this process.

Displaying a CSV File

As for all linked files, the CSV will display at the end of the section if not hidden on request. As a CSV is a representation of table data it is parsed and displayed as a table. If the data does not display correctly then it also will not process correctly, so scan the table when adding or changing data.

Editing a CSV File

Clicking the link or double-clicking the contents will enter edit mode using a text editor. A text editor provides more flexibility than a table editor.

CSV Format

The default format consists of one title line, comma separators and single or double quotes. Multiple lines are allowed in quoted variables. Spaces outsided of quoted variables will be trimmed - left and right. 

Other Formats

Formatting expectations can be changed on page setup, section setup or for a specific data load.

CSV Processing

All scripts have access to a csv processor. Just put dsl.csv in your page Setup script. Check the footer of this page for an example.
