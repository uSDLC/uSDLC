Screencasts

for talks and web walk-throughs

The Screencast DSL is designed to demonstrate uSDLC techniques from within this common framework. It is creating a DSL to simplify the use of WebDriver for screencasting.

Associate a DSL name with a backing script provider.
Open a pop-up window that is both driver and prompt display.

Each command will end in a text field that will be displayed as a note or prompt..

Execution pauses until continue is requested. The current location is checked to make sure it has not changed before continuing.
Create a screencast as a new named page in the Sandbox. Requires title, subtitle and synopsis.
Add a footer that contains links to Setup, Cleanup. Code to be executed before and after an instrumentation run. Requires language extension. (groovy, java, coffee, ...). Add link to parent page.
Create a new section at the end (but above footer). Requires a list of paragraphs to insert in the section.
Select a section for processing based on some text in the section.
Select next section.
Convert a string in the current section into a link. Requires text, link type (html, groovy, ...) and option to pause during steps.
Go to child pages based on link name.
Edit activation scripts based on name.
Bring up context menu and run a command chain.
