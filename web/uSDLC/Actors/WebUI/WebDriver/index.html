Web Driver

The New Age Browser Driver

Selenium WebDriver will allow an application to manipulate a browser without the need if an intervening proxy server.

Configuration

WebDriver configuration is in /web/Options/Configuration/webDriver.groovy. Note that the browsers that WebDriver support are being updated continuously. Download them from http://code.google.com/p/selenium/. Download drivers for additional environments and put them into /web/lib/jars. In some cases, such as Chrome in OS X, you will also need to download a binary. Not all drivers work seamlessly on all operating systems. I find that either firefox or chrome has worked in all cases. Set the default in the same configuration file. Typically code will use one of the real browsers during development and htmlunit for continuous integration testing.

Setting the Driver

Normally once you have a driver running on an installation, you will stick to it. Sometimes it is necessary to test more than one client - or switch to a headless htmlunit driver.

Opening a Web Page

In most applications of WebDriver, a specific site will be under review. One approach is to create a browse command in the application DSL. For our examples here, we will be driving uSDLC itself.

Checking Page Contents

First we need to find something on the page. Use id, name, link text, css selector, xpath, class name, partial link text, regular expression or let uSDLC take an educated guess. The same procedure is used by checking, mouse clicks and data entry. Use more exacting methods, suck as a css selector for selecting elements on the page that are not form related.

  • check target, contents -- is a shortcut to 'check all'.
  • check all target, contents -- The contents of a text box must exactly match. For select boxes or tables, the contents can be an array - and all elements of the array must be present or selected.
  • check some target, contents -- The contents of a text box must include the string. For select boxes or tables, the contents can be an array - and at least one element of the array must be present or selected.
  • check none target, contents -- The contents of a text box must not include the string. For select boxes or tables, the contents can be an array - none of the elements of the array must be present or selected.
  • check empty target -- The contents of a text box must include the string. For select boxes or tables, the contents can be an array - and at least one elements of the array must be present or selected.
  • check selected target, list -- Specifically for <select> elements. All the items and only those items must be selected.

The content is usually a string or array of strings. Sometimes, more accuracy is required. For these, use regular expressions instead by replacing the quotes with slashes.

Mouse Navigation

Specify what you want to click on. Use -> between selectors to drill through pages to the one required.

Data Entry

The primary form of data entry is to send keys to the browser - just as if they were typed. Special routines handle lists, check boxes and radio buttons. All are driven by enter. Fields are changed with name/value pairs - where the name can be any of the options described above. If the page contains more than one form, include a form entrie with the value being the form id.

Find by Label

Previous checks, clicks or data entry have used investigation of the html source to clearly specify the element to be found. In practice while presenting pages to domain experts it would be clearer to use visible elements on the page. Potential matches can be found by:

  • Label text
  • Table column or row title
  • Field set legend
  • Text inside the same elements as an input element

The label comparison is a reasonable sub-set of the visible text. It can also be a regular expression. The form name is equally flexible - being any text (or regular expression match) inside or wrapping the form. So you can specify a form by choosing a unique element in it.

Tables

Data if often displayed in tables. Use the check command to see if a row or column contains, or does not contain, data.
