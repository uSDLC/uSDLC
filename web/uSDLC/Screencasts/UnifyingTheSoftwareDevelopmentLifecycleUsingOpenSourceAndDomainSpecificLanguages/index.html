  • I am a software designer with over 30 years experience in diverse fields including mining, manufacturing, defence, tolling and government. I have taken "time off" to work as a software architect and software manager implementing agile practices in hostile environments. My frustration with the lack of cohesiveness in the development lifecycle has lead to a passion for improvement in this area.
  • Unifying the Software Development Lifecycle Using Open Source and Domain Specific Languages
  • Open source technologies are now up to the challenge of unifying the software development lifecycle. In this talk I will demonstate how interactive documentation and domain specific languages allow the use of open source systems to support all teams in a project - from client to support. I will be introducing usdlc.net - an open source interactive documentation system that supports this concept for small to medium teams. 

    First half is about how an interactive documentation can help all groups involved in a software development project. The second half is a hands-on demonstration of designing and implementing a real-world sub-system using this approach. It uses the open source uSDLC framework with Java and Groovy on the jvm. This is the debut for uSDLC. uSDLC.net will be live for the conference. usdlc/usdlc is available on github now.

Introduction

  • uSDLC is an open source project.
  • I have spent years frustrated by the poor communication between different facets of the SDLC.
    • As a developer, knowing that the BAs had gathered information for writing their documentation that I could use in writing better tests.
    • As an architect trying to make reality match the specification or tender - and knowing any document I write will be so out of date as to be of little use in later phases of the SDLC.
    • As a test manager trying to understand why the software developed does not match the specification required - knowing that there are good reasons but not knowing what they are.
    • In support having to read uncommented code because I can't trust design artifacts to match the product.
