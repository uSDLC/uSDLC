Dependencies

Code, Libraries and Sub-systems we can use

uSDLC can use any of a large number of systems to integrate development and the rest of the SDLC. If your code is dependent on one of these systems, you can use these pages to get them and keep them up to date. JVM based dependencies, including source and documentation are mostly serviced from Maven repositories with Ivy. Other dependecies are downloaded and installed directly.

If you attempt to run an actor that has a registered dependency that has not yet been initialised, you will get a pop-up with a link to load it (and it's dependencies) into the system.

To update dependencies, click on the spanner on the tool-bar and select the page of the same name. You can run the page to update all dependencies  that are marked as required or previously loaded. You can also load new dependencies by running their link.

There are dependency section templates to ease the adding of new sources. Right-click on a blank section and select Template / Dependency Type. Existing dependency sections can be updated the same way.

Downloading dependencies can take quite some time. uSDLC does provide a feedback bar to provide some information on progress.
