Instrumentation

Instrumentation is the glue between definition and implementation. It is readable by the audience for the page and by the computer. It also replaces or wraps traditional technical test code.

Definition

The long and short of it is that instrumentation is a script written in a computer language that is palatable to the readers of the page. Low level technical pages will have instrumentation scripts readable by developers - often in the same computer language as the target system. High level pages will use more natural language like domain specific languages readable by domain specialists who are likely not to be software developers.

Creation

A section can create a reference to a child page in one of three ways:

  1. Select New | Instrumentation from the menu (or type NI)
  2. Select Section | Insert | Above or Below from the menu (type SIA, SIB or Ctrl-I, Crtl-A) and choose the second tab, labelled Instrumentation.
  3. Add a link using the editor on an existing section. The last radio button is instrumentation.

Instrumentation Type

As has been traditional forever in the software community, the file type is defined by the file name extension - the word after the final dot.

uSDLC supports a growing list of computer languages using their standard extensions - java, groovy, js, coffee, ... In addition, domain specific alternatives can also use special extensions - screencast, medical, accounts, ...

Select an existing extension from the drop-down, or add a new one following a stop in the title for the form approach and the link name if adding instrumentation from the editor.
