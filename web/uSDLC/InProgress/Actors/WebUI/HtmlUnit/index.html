HtmlUnit

Testing with a virtual browser

HtmlUnit runs as a mock browser that purports to do all that the real thing does. You drive it with a simple DSL describing the actions you want to perform.

This first test is simplified - just making sure we can click on a section of this document and make it highlight..

A web application will provide a consistent interface. It makes sense to create classes with consistent HtmlUnit actions for re-use by actors. For uSDLC there is one called Sandbox. It can be used to manipulate uSDLC pages created and worked within the Sandbox folder. Here we open a page called scratch and create a new section. Check that the HTML editor has been opened.

HtmlUnit is available as drivers for both Selenium and Geb.
