Screencasts

for talks and web walk-throughs

The Screencast DSL is designed to demonstrate uSDLC techniques from within this common framework. It is creating a DSL to simplify the use of WebDriver for screencasting.

Associate a DSL name with a backing script provider.

Open a pop-up window that is both driver and prompt display.

Create a screencast

as a new named page in the Sandbox. Requires title, subtitle and synopsis.

Note or Prompt

Each command will end in a text field that will be displayed as a note or prompt..

current location

Execution pauses until continue is requested. The current location is checked to make sure it has not changed before continuing. Ordinarily a prompt is wrapped in a snap and recover. Here they are called explicitly to highlight the functionality.

Create a new section

at the end (but above footer). Requires a heading and a list of paragraphs to append a section or insert above the one in focus. Existing sections can be selected by contents or relative position and checked to contain contents.

Links

Convert a string in the current section into a link. Requires text, link type (html, groovy, ...), source and an option to pause during steps. If the current section does not have the text supplied, search other sections for it.

Steps

Instruction speed can be controlled between prompts to provide more visual feedback. Use step with fast, slow as the number of seconds.

Go to child pages based on link name.

Edit activation scripts based on name.

Bring up context menu and run a command chain.
