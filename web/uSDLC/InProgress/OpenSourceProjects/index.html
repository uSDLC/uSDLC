Open Source Projects

How does uSDLC fit in?

Open source projects still have a SDLC. Many have distributed teams and are open to contributions from outside the traditional team. uSDLC has a lot to contribute to making open source projects work more quickly and become more palatable to potential consumers.

uSDLC is open source - using the Apache 2 license.

uSDLC uses many open source components - some of which are listed here:

  • Ckeditor - a wysiwig section editor anyone can use
  • CodeMirror - for in-page text editing editing
  • Coffeescript - advanced language - compiles to Javascript
  • Geb - very groovy browser automation
  • Groovy - the uSDLC server is written in Groovy
  • javascript - Find a definitive link for this pervasive language if you can
  • H2 - Fast SQL with compatability modes for most mainstream databases
  • HtmlUnit - headerless browser for automated testing
  • Ivy - The Apache dependency manager
  • jQuery - the write less, do more javascript library
  • OpenCSV - processing tabular data
  • Rhino - The Mozilla JVM based Javascript engine
  • Selenium - split personality web browser automation
  • Scala - where functional and OO meet
