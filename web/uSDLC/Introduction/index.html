Introduction

In the beginning

We, as a group, are doing a good job of putting processes in place that improve the performance and reliability of phases in the Software Development Life-cycle. The communications between the phases, however are typically unwieldy, expensive and ineffective. Usdlc is an attempt to address these issues by integrating best-of breed processes from the different phases in a platform that provides facilities for better communication.

Modern software development methodologies, techniques and technologies are now ready to support the unification of the software development life-cycle. This system works step-by-step towards that goal. While the term SDLC is as old as the IT industry, it is still a valid term for encompassing all the processes that are required to create and maintain a software system. This system uses agile methodologies to build upon the Lean principals to reduce waste and increase productivity.

uSDLC is a browser based distributed system that can be used by all participants of a software development project. It provides integration between groups and stages within the SDLC.

  1. Provide a framework to support Behaviour Driven Development (BDD) across the SDLC.
  2. Embrace Lean principles.
  3. Support agile processes.
  4. Introduce documentation to the agile and lean processes in a way that provides benefit without introducing waste.
  5. Leverage best-of-breed systems for better communication between the steps of the software development life-cycle.
  6. Minimise the waste that is incurred by loss of information between the SDLC steps.
  7. Integrate tools that are used because the benefits out-weigh the effort involved in using them.
  8. Remove the need for dead-end artifacts such as stand-alone design documents and wire-frame screen designs.
  9. Multiple instances awareness - for both information sharing and remote actioning.
  10. Generate reports that are of value to all participants.
