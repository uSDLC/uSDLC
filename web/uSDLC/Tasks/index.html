Tasks

What to do, what to do...

Every page, section or actor link is a task. Tasks are used to track and assign work.

Task Report Creation

uSDLC comes equipped with a Task Report Actor. You generate a report from a tab on the create link box - available from the link button on the tool-bar. From here you decide what task states to display.

While is it possible to generate report pages anywhere, convention is to put them in the home pages chapter. When a member is added to the system they are given a home page from a configuration template. In the standard configuration this template includes a link to reports relevant to them member - tasks assigned to them, tasks completed, etc.

A project manager or team leader may add reports that represent their interest areas. The members area also includes a member called Project. Here resides reports of general interest.

Task State

The list of states can be changed from the configuration page. By default uSDLC has the following states:

  1. Undocumented - as in a page without sections or a section without content.
  2. Incomplete - tagged for sections that do not have links to actors to connect passive and active components of the project.
  3. Unassigned - a section that is document complete but never activated.
  4. Assigned - a section in process for implementation.
  5. Ready - tasks ready to activate
  6. Failed - tasks where activation has failed
  7. Orphaned - pages and actors that do not have a link in the document
