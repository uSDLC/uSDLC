Active Mode

Integration by Activity

Pages can be in edit or run mode. The latter activates links and commands that connect the documentation context to the system under review.

Links

Most of the activity that constitutes a page run happens through links. Links can be to other pages, scripts or data.

Absolute Links

Absolute links - whether to another site or within uSDLC are treated like any other web application - click on them to follow them. They are not followed as part of the run. When running this page, neither of the two links in this paragraph should cause an error.

Page Links

Page links end in .html or specify the domain. A page link can also be generated with code by having an additional server extension. index.html.groovy will expect the matching groovy code to produce HTML to populate the page content. Page link decorations will match the state of pages and links of children - being purple if not hooked up, red if a child has not passed or green otherwise.

Relative Links

Relative links do not have a domain and do not start with a slash (/). They are followed, and the contents run. This link is a page that should mark as a pass during a test run. Note that it is created by clicking the link icon on the editor tool-bar. Leave the type as local and enter the name as a sentence. For this example, Relative links was typed in. On save this translates to ./RelativeLinks/index.html. If you want to specify a different directory or a file in the current directory, specify it fulling (as in RelativeLinks.html or myTests/RelativeLinks.html)

Actors

uSDLC supports actors numerous scripting languages. Links to scripts are of the form name.client.server so that we can define action on both sides of the divide. A file myScript.js.groovy will run a groovy script that is expected to send javascript back to the server. To produce a script that responds with HTML, either don't give it a client extension or use .htm.

Actor Link Decoration

When you create a link to an actor you will see a purple underline. This indicates that the underlying actor has not been created. Once it has been created it will have a red underline until it returns a success. Currently successful links will have a green underline. Attempting to run a script that does not exist will raise a failure.

Script Activation

Clicking on a actor link will cause the script to run. To edit the script instead, hold the Alt key down before selecting the link. Hovering over a link will provide reminder text for this functionality.

Actor Parameters

Links can provide script specific parameters. Other parameters are provided by the environment. All scripts are provided a parameter called id that uniquely specifies the source for the link in the form page.path.s1a1, being the path to the page followed by the section number and link number within the section.

The event parameter defines how the actor was triggered - pageLoad, run or click.

In-Line Actor Display

If a section consists of only one link and that to a script, the source to the script is displayed on the link page. This is useful for software systems documentation where the source is integral to the book.

Editing an Actor Script

For developers we recommend adding uSDLC to your IDE project. That way you can make use of the advanced IDE editing features. For quick changes or tests outside a development environment, uSDLC integrates a source code editor, EditArea. For many languages, auto-formatting and code completion are turned on.

Actor Input

A free-standing actor will be passed the content of the surrounding section as input.

Actor Output

Actors provide some sort of feedback. They can act as sources for other scripts, provide pass of fail behaviour or display information to the user. In most cases this feedback only becomes available once the script has completed operations and sent a response to the browser. Longer running scripts can also provide progress information by writing to standard error. If this information is detected, it will be displayed on the uSDLC console - as set to a window or frame that appears during the run and is hidden again once the script has completed.

Free-Standing Actors
