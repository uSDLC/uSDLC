Active Mode

Integration by Activity

Pages can be in edit or run mode. The latter activates links and commands that connect the documentation context to the system under review.

Links

Most of the activity that constitutes a page run happens through links. Links can be to other pages, scripts or data.

Absolute Links

Absolute links - whether to another site or within uSDLC are treated like any other web application - click on them to follow them. They are not followed as part of the run. When running this page, neither of the two links in this paragraph should cause an error.

Page Links

Page links end in .html or specify the domain. A page link can also be generated with code by having an additional server extension. index.html.groovy will expect the matching groovy code to produce HTML to populate the page content.

Relative Links

Relative links do not have a domain and do not start with a slash (/). They are followed, and the contents run. This link is a page that should mark as a pass during a test run. Note that it is created by clicking the link icon on the editor tool-bar. Leave the type as local and enter the name as a sentence. For this example, Relative links was typed in. On save this translates to ./RelativeLinks/index.html. If you want to specify a different directory or a file in the current directory, specify it fulling (as in RelativeLinks.html or myTests/RelativeLinks.html)

Scripts

uSDLC supports numerous scripting languages. Links to scripts are of the form name.client.server so that we can define action on both sides of the divide. A file myScript.js.groovy will run a groovy script that is expected to send javascript back to the server. To produce a script that responds with HTML, either don't give it a client extension or use .htm. Scripts can be free-standing, producers or consumers.

Free-Standing Scripts

Any script not in a table is run as a free-standing script. It is executed and the results returned to the browser in a form defined in the client extension. This script will change the page heading to red. This script is purely client-side and will check to make sure the heading is red.

Most scripts will call a server-side actor to interact with the system under review. Once a script has been run any responses will display on a pop-up window. Clicking on a script that has already been run will redisplay the results window. To run again go to edit mode and back to run mode. This script is a groovy actor that will display a simple message. Note that the link changes colour during and after the script has been run.

Producers

Scripts that returns content (htm, xml, txt, etc) are producers. For free-standing scripts, this is only visual - hover over the link after a run to see the results. A .htm result can also be active in that script tags are run. This is the normal type of result returned by tests.

More commonly producers are in tables - other than column 1 if it is a heading column (reserved for consumers).

Consumers

If a script is in column one of a table and that is a heading column it is treated as a consumer. The scripts in the other columns of the same row are executed first and their output is collected and sent to the consumer as the body of a post. Any text in producer cells is sent as well.
