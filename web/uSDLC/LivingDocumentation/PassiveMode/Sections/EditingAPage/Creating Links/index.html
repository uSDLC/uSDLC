Creating Links

to glue the document together

To create a link to a child page, use the link tool on the editor bar.

Use the local link protocol for D3 internal operations. When you highlight text it will use this to fill the address box. The radio buttons below the name field select the link type - child page, actor, etc. If an actor is chosen you can select the language/system to use. D3  will remember your selection.

The editor link menu defaults to local - the mode to use to navigate internally within the document. For child pages, just copy and paste the link text into the URL field. uSDLC will take the effort to convert this into a readable link and connected page. It removes spaces from the link url, creates a matching path and prepares for an index.html for page content.

If a page in the same directory as the parent is required, add .html to the end of the link name.

You can refer to pages at other locations in the document by giving a full url relative to the domain (being one that starts with /). These can be obtains by copying direct links to these pages from their parent.

To open other documents or any web page, use a full url - one starting with http:// or https://.

Links are also used to reference actors - the code modules used to activate a uSDLC document. An actor has an extension that indicates the language to be used - as in .java or .groovy. Actors have have another extension indicating the client presentation. The form of an actor url is name.clientExt.serverExt. If the client extension is .html, the page is treated and document content - otherwise as actions.
