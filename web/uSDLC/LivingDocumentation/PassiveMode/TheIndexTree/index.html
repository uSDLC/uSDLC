The Index Tree

Hitch-hiker's Guide

In the standard layout, the index is to the left of the contents. It contants links to every page in the document allowing for direct navitation.

Every page in a uSDLC document that is referenced in a parent document will be displayed in the index tree in the order they appear on the parent page.

Removing the link to a page will make it inaccessable from the document. The page will still exist in the document and be opened by direct url - or by adding a link again.

Because the index tree is dynamically loaded, no unopened branch knows whether it has any children. When you open a branch, it's children are displayed if they exist - otherwise the opening handle is greyed out.

As you move, cut and paste sections in a document that hold links, the index will update so that the index will reflect the document structure every time updates occur.

When you open or close the tree, this status is saved by the browser. On re-opening the page you will be presented with the same structure and active page as when you last used it on this machine. The page open for the last session will re-open and be highlighed in the index.
