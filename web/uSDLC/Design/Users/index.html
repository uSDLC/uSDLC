Users

User authorisation, authentication, managment and support.

User Definition Page

uSDLC includes a document based user system. Create one page per user per project in Environment / Users. These user pages reference any number of scripts that can be used to set up the user environment. The one called 'Password' will be rewritten whenever the user asks to change password. Typically the rest of the scripts will indicate the groups the user belongs to - using regular expressions.

User Management DSL

When a user logs in the scripts on their definition page are run. These scripts first load this dsl to provide user management functionality.

User Authorisation

For a common access uSDLC, the default user is 'Guest'. A user who has an definition page can log in using the menu 'User | Log in'. Logging out will reduce access back to 'Guest'.

For a personal copy of uSDLC, the preferred approach, setting userId in the configuration will cause an automatic log-in. The easiest way is to edit the startup script and change the userId parameter.
