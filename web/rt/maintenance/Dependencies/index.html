Dependencies

Dependent children that work - an oxymoron?

uSLDC will work in it's base state if all actors are written in Groovy or Java. To environments where other computer languages are king, we need to download the optional dependencies. Most jvm dependencies come from maven repositories. For other parts of the world, downloads and installation  can be either automatic or prompted.

HtmlUnit tests web pages in a headerless fashion (without a browser). This is a necessary module if your project has a web/browser based user interface.

 

Apache Ivy

Ivy is a configuration management tool. One could say that it is the configuration tool for the JVM. Maven and Grape rely on it to download and install files. uSDLC requires it to download optional JVM based components as well as keep the required ones up to date.
