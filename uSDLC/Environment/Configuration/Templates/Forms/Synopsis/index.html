When you open a page for the first time uSDLC offers advice on filling it the synopsis that can be seen from the parent page.

Synopsis: General

Notes: 
Enter the synopsis for this general unclassified page. Accept the responsibility to put in more detail because this page type does not lie within the formal design by decomposition guidelines.

Synopsis: Project

Notes: 
Enter a complete but concise overview of the project. If you feel the need to elucidate, create a guide section below the synopsis. Sections on this page are usually vision points.

Synopsis: Vision

Notes: 
Vision clarification and recording is highly individual.  One beneficial approach is decomposition. When describing a vision to others you will use a number of key and telling points. A vision page is for one of these points.

Synopsis: Goal

Notes: 
All stakeholders will have goals. Most can be tied directly to the vision, but some (such as the system must be secure) are independently driven. Typically there will be a catch-all vision statement on a project page called "Meet stakeholder independent requirements".

Synopsis: Capability

Notes: 
Capabilities are about what the software needs to do, while features are how it will do it.
