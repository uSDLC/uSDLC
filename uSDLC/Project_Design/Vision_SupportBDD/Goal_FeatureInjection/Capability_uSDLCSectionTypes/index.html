Modelling with Page Types

Each project starts with a page of that type. The main project may include other project pages at any level. These sub-projects are separated by time, functionality or vision.

Once you have uncovered the true business values, they become vision points on the project page. Each vision point becomes a page of goals to reach that vision. Each goal can be described in a list of capabilities. A capability can be decomposed into a list of features, while features can be described by examples (scenarios).

I call this design by decomposition. It depends on the size of the project and the composition of the stake-holders on whether all these levels are necessary. The important point is that we start with business value and drill down to working examples. In this way the vision holder can see where requirements meet product and the developer can get context when instrumenting and coding for the examples.
