Installing USDLC

Step one: download, install and test uSDLC. uSDLC is a JVM based client-server application. The client is any modern browser. The server is designed to be run from a single download without additional installation.
Guide

Prerequisites

The only prerequisite for uSDLC is a Java runtime (JVM). Bruce is running OS X - which comes with Java pre-installed.

Guide

Download

Bruce has a choice. He can download a single-file install from usdlc.net or he can clone the project from github. The results are identical, but as Bruce is not a Java developer and he does not regularly use GitHub, he finds it easier to download uSDLC-full.jar.

Bruce keeps his project source code in directories under ~/dev, so he creates a directory ~/dev/uSDLC and copies the jar to it.

Guide

Updates

Any time Bruce wants to update uSDLC to the latest version he downloads a new copy of uSDLC-full.jar and overwrites the one in ~/dev/uSDLC. When run it will unpack and update only those files that are newer.

Guide

First Run

Apple has an effective jar launcher, so all Bruce has to do is go to ~/dev/uSDLC in Finder and double-click uSDLC-full.har. He will see the directory fill up with all the working files. Then the default browser will open to the local uSDLC home-page. If Bruce wants to see more information he can open the OS X console (/Applications/Utilities/Console.app).

Guide

Automatic launch

Again OS X makes this easy. uSDLC will be on the dock with the default Java icon. Bruce right-clicks on it and selects the menu Options | Open at Login.
