Guide

Not many software systems are so self-evident that users won't occasionally need help to understand how something works.

The best way to learn a new system is by example. Fortunately as we are designing by example, we can use many of the same cases to provide a user friendly guide to the system.

Once a user has become familiar with the system, the most often need documentation to tell them how to do something new.

Lastly we can never completely remove the need for a dry reference document.
