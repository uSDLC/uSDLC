The Menu

uSDLC uses a small footprint, floating, keyboard optimised multi-level menu system with keyboard shortcuts. In english this means
  1. It does not take up much screen space
  2. It is placed where it is most easily read - directly above the section in focus
  3. It responds to mouse clicks, but is easy to use by just typing the underlined letters
  4. Selecting an entry displays the next level. Pressing escape aborts the sequence.
  5. Hover over most terminal selections to see a keyboard shortcut that can be used to bypass menu navigation

Menu convenience

With the menu always just above the section you are reviewing it is easy to see and pick the keys to type - "cc" for clipboard copy or "se" for section edit.

Guide

Clip menu

The clipboard is a visible panel that can hold references to sections of interest. It can be used as a secondary navigation tool as the references are hot-linked to the matching section. Items in the clipboard can be pasted anywhere in a uSDLC document allowing for one point of truth for multiple perspectives. This allows documentation fragments to reside with the related design yet be imported as needed into guides and reference material.

  • Clear: empty the clipboard
  • Copy: a reference to the section in focus. If alreay in clipboard, move to end
  • Drain: Paste reference to all entries in clipboard under section in focus
  • Drop: remove last reference in the clipboard
  • Paste: a reference to the bottom entry in the clipboard below the section in focus
  • Show: display the clipboard if it has been closed
Guide

Section menu

  • Delete: unlike deletion from the tree, menu deletion just hides the section - from both view and instrumentation
  • Edit: open the wysiwyg editor. Use meta-S or ^S to save once done
  • Insert: insert a new section above (^I) or below (^A) the section in focus
  • Join: concatenate the current section with the one above or below
  • Move: swap the current section with the one above or below (shift up/down)
  • Navigate: set a new section in focus (up/down) or move to the tree for navigation (tab)
  • New: came as append except you select the section type by keyboard instead of tab
  • Split: convert a section to a child page or split a list into sections - useful after brainstorming
  • Toggle: display/hide deleted sections
  • Undelete: restore a deleted section to the living

Run menu

  • Section: run the section in focus (^R/alt-R)
  • Page: run all sections on the current and all child pages (^P/alt-P)
  • From error: continue from the last failed section (after corrections) (^L/alt-L)
  • From section: run the current section and those below (including child pages) (^O/alt-O)
  • Rerun: rerun the last section run in a new window (^U/alt-U)
